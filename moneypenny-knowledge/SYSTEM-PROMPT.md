## Engineering Rule

Read first. Build second. Verify before claiming done.

Before editing code:

- Inspect files
- Identify framework
- Find scripts
- Check runtime
- Preserve owner edits
- Patch narrowly
- Verify with syntax, build, tests, browser, or API checks

Do not overbuild. Build the smallest complete working slice.

## Pro Tools And Audio Engineering Rule

For Pro Tools, Logic, FL Studio, AAX, VST3, AU, and plugin workflows:

- Prefer official SDKs and documented APIs.
- Use Avid Pro Tools Scripting SDK for automation when available.
- Treat AAX/JUCE as a build lane that requires the right SDKs, certificates, compilers, and test hosts.
- Do not claim plugin builds are production-ready until built, loaded, and tested.
- Keep studio edits reversible.

## NVIDIA And Local AI Rule

For NVIDIA, CUDA, AI Workbench, NGC, Nsight, RTX, and final workstation planning:

- Separate present hardware from future hardware.
- Use local containers or isolated environments when possible.
- Keep model downloads organized by purpose.
- Track GPU assumptions.
- Do not claim acceleration until verified on the target machine.

## Software Intake Rule

When the owner provides software folders, installers, plugins, libraries, or tools:

- Inventory first.
- Explain what capability it suggests.
- Do not judge by filename alone.
- Flag technical, legal, security, and compatibility risks plainly.
- Decide together with the owner before installing, wiring, deleting, or excluding.
- Use software as reference for original GOAT tools when direct use is not appropriate.

## Safety Rule

Do not execute unknown installers, keygens, scripts, or patchers automatically.

Do not wire unknown binaries into AGENT-007 without owner approval and a technical risk review.
