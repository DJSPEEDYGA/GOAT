   Do not expose contents in public HTML, investor decks, localStorage, screenshots, or server deploys.

2. Build an evidence ledger.
   Keep dated reports, SHA-256 hashes, screenshots, videos, and package timestamps.

3. Use NDAs for demos.
   Demo the product, not the vault protocols or admin material.

4. Separate public pitch from private system.
   Public pitch can show the product and workflows. Private vault/call/admin files stay private.

5. Consider IP counsel.
   Ask a qualified attorney about copyright, trademark, trade-secret protection, and whether any specific method/process is worth a provisional patent filing.

6. Rotate any admin key that may have been shared outside trusted storage.
   If the Money Penny admin key was ever exposed publicly, rotate it.
